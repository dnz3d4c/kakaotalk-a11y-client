## 왜 만들었나

저는 시각장애인 사용자인데요, 카카오톡 쓰면서 불편한 게 좀 있었어요.

NVDA용 카카오톡 추가 기능이 있긴 한데, 뭔가 아쉬웠어요. 특히 말풍선 공감 이모지는 스크린 리더로 아예 접근이 안 돼서, 하트 하나 눌러주고 싶어도 방법이 없었거든요.

공감 이모지가 클릭이 안 되니까 어떻게 개선할 수 있을지 고민만 하다가, 문득 아이디어가 떠올라서 만들기 시작했어요. 이미지 인식으로 시도해봤더니 되더라고요. 이거 좀 확장하면 쓸만하겠다 싶어서, 그렇게 시작한 게 여기까지 왔네요.

지금은 이런 것들이 돼요:
- 친구탭/채팅탭 탐색
- 채팅방 메시지 읽기
- 말풍선 공감 이모지 클릭

제가 쓰려고 만든 건데, 같은 불편함 겪는 분들한테도 도움이 되면 좋겠어요.

## 기술적 접근 방식
이미지 인식(템플릿 매칭) + 좌표 클릭 방식 채택

### 왜 이 방식인가?
1. UI Automation API: 카카오톡이 접근성 정보를 노출 안 해서 사용 불가
2. 이미지 인식: 접근성 정보 없어도 동작 가능

## 카카오톡 공감 기능 스펙

### 이모지 종류 (공식 6종)
1. ❤️ 하트 (좋음)
2. 👍 엄지척 (최고)
3. ☑️ 체크 (확인)
4. 😄 웃음 (기쁨)
5. 😮 놀람
6. 😢 슬픔

### 동작 방식
- PC에서 말풍선 우클릭 → 공감 이모지 6개 표시 → 클릭하여 반응
- 이미 달린 공감은 말풍선 아래에 표시됨
- 같은 이모지 다시 클릭 시 취소
- 7일 지난 메시지에는 공감 불가

### 시각적 특성
- 이모지 크기: 약 18-24px
- 위치: 말풍선 바로 아래, 가로로 나란히 정렬
- 내 메시지(노란 말풍선): 오른쪽 정렬
- 상대 메시지(흰 말풍선): 왼쪽 정렬

## 핵심 요구사항

### 필수
1. **100% 로컬 처리** - 외부 서버 통신 절대 불가, 개인정보 유출 방지
2. **NVDA 스크린 리더 호환** - 모든 UI 요소에 접근성 정보 제공
3. **키보드 단축키 지원** - 마우스 없이 조작 가능

### 기능
1. 화면에서 카카오톡 공감 이모지 영역 자동 탐지
2. 탐지된 이모지 목록을 음성으로 안내 (예: "하트 1개, 엄지 2개")
3. 원하는 이모지 선택하여 클릭

## 고려 사항

### 변수
- 라이트모드 vs 다크모드: 배경색 다름
- 1:1 채팅 vs 그룹채팅: 숫자 표시 여부 다름
- 창 크기/위치 변경 시 좌표 달라짐

### MVP 범위
- 라이트모드, 1:1 채팅 먼저 지원
- 추후 다크모드, 그룹채팅 확장

## 이모지 템플릿
`src/kakaotalk_a11y_client/emojis/` 폴더에 포함 (빌드 시 패키지에 포함):
- `emoji_heart.png`: 하트 이모지
- `emoji_thumbsup.png`: 엄지척 이모지
- `emoji_check.png`: 체크 이모지
- `emoji_smile.png`: 웃음 이모지

## 참고
- 사용자는 전맹 시각장애인
- 이모지 직접 크롭 불가 → 자동 크롭 로직 필요
- 템플릿 이미지는 사용자가 제공한 스크린샷에서 자동 추출
